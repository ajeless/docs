# Documentation Overview

This repository is intentionally structured to separate **vision**, **working models**, **speculative ideas**, **research**, and **actual decisions**.

That separation matters because the project is still in the ideation phase. The goal is to preserve nuance without accidentally promoting every interesting thought into architecture.

## Suggested reading order

If you are new to the project, read in this order:

1. [`../README.md`](../README.md)  
   The shortest overview of the project and its current status.

2. [`vision.md`](vision.md)  
   Why this project exists and what it is trying to become.

3. [`principles.md`](principles.md)  
   Guardrails that keep the project coherent and grounded.

4. [`mental-model.md`](mental-model.md)  
   The current conceptual framing: programmable reality, spatial OS, layered services.

5. [`feasibility-boundary.md`](feasibility-boundary.md)  
   What the platform can directly implement, what it can scaffold, and what must emerge.

6. [`open-questions.md`](open-questions.md)  
   The most important unresolved questions.

7. [`ontology.md`](ontology.md)  
   Draft vocabulary for describing the world in a consistent way.

8. [`research-topics.md`](research-topics.md)  
   Technical and conceptual research fronts.

## How to use this repo

- Put stable overview material in `README.md` or `docs/`.
- Put rawer, more speculative thinking in `docs/ideas/`.
- Put actual commitments in `docs/decisions/`.
- Avoid treating attractive terminology as settled architecture unless the repo explicitly says so.

## A note on certainty

Many terms in this repo are **working terms**, not final names:

- “Programmable Reality”
- “Spatial OS”
- “Reality Kernel”
- “Information Layer”
- “Persistence Layer”
- “Agent Orchestration Layer”

These help structure discussion, but the project is still exploring what is metaphor, what is model, and what will eventually become implementation.
