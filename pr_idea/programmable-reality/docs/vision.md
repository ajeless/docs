# Vision

## Short version

Programmable Reality is an open-source, community-driven platform for creating and modding interactive worlds where authors can configure not only content, but aspects of reality itself: scale, simulation depth, information behavior, persistence, and modifiability.

## Why this project exists

Most existing engines are optimized for making games inside a relatively fixed set of assumptions:

- a scene or level is the fundamental unit
- simulation depth is limited or specialized
- persistence is either bolted on or game-specific
- scale changes often require different architectural choices
- modding is frequently secondary to shipping the base product

This project explores a different starting point.

Instead of asking, “How do we make one game?” or even “How do we make a better engine?”, it asks:

**How do we create a shared substrate on top of which many different kinds of realities can be authored?**

That substrate should support:

- small local co-op experiences
- tactical or systemic simulations
- highly authored scenarios
- moddable sandbox worlds
- experimental realities with unusual rules
- eventually, persistent community-run worlds

## The central shift

The most important conceptual shift in this project is from **game engine** to **programmable reality**.

That shift matters because it changes the target of design.

The goal is not only to render graphics or execute game logic. The goal is to let creators shape:

- how space is represented
- how information propagates
- how simulation is executed or abstracted
- how memory and persistence behave
- how deeply a world can be modified
- how grounded or fantastical a world should feel

## What success would look like

Success does not necessarily mean “one massive persistent universe.”

Success would mean a platform where the same conceptual framework can support realities as different as:

- a small throwaway co-op mission with minimal persistence
- a tightly tuned tactical sim with high determinism
- an immersive seamless world with deep systems
- an authored world with surreal or impossible rules
- a community-hosted persistent environment with evolving institutions

## What is special here

The distinctive ambition is not only scale.

It is the combination of:

- scale where needed
- local simplicity where possible
- first-class modding
- configurable realism
- configurable persistence
- configurable information rules
- room for both strict systems and emergent behavior
- a platform that stays open to deep forking

## What this project is not trying to do

This is not a metaverse pitch, and it is not a claim that software can fully encode human society or meaning.

It is also not, at this stage, a settled architecture.

Right now the project is trying to become intellectually coherent before it becomes technically committed.

## Near-term purpose of the repo

In the near term, this repo exists to do four things well:

1. stabilize language
2. preserve nuance
3. separate ideas from decisions
4. create a research roadmap that future humans and coding agents can build from
